# 微信小程序项目结构说明

## 一、目录结构概览

```
wechat_mini_demo/
├── project.config.json    # 微信开发者工具项目配置
├── app.json              # 小程序全局配置
├── app.js                # 小程序全局逻辑
├── app.wxss              # 小程序全局样式
├── pages/                # 页面目录
│   └── index/            # 首页模块
│       ├── index.js      # 页面逻辑
│       ├── index.json    # 页面配置
│       ├── index.wxml    # 页面结构
│       └── index.wxss    # 页面样式
└── tutorial/             # 教程文档
    ├── 01-项目结构说明.md
    ├── 02-配置文件详解.md
    ├── 03-页面开发指南.md
    └── 04-开发工具使用.md
```

## 二、各目录和文件的作用

### 1. 根目录文件

#### project.config.json
- **作用**：微信开发者工具的项目配置文件
- **包含内容**：
  - 项目名称
  - AppID（如果没有可以为空）
  - 编译设置
  - 编辑器配置
  - 版本库设置

#### app.json
- **作用**：小程序的全局配置文件
- **包含内容**：
  - 页面路径配置（pages）
  - 窗口外观配置（window）
  - 标签页配置（tabBar）
  - 网络超时配置（networkTimeout）
  - 调试模式配置（debug）

#### app.js
- **作用**：小程序的全局逻辑文件
- **包含内容**：
  - App() 构造函数
  - 全局生命周期函数（onLaunch, onShow, onHide）
  - 全局错误处理函数（onError）
  - 全局数据（globalData）

#### app.wxss
- **作用**：小程序的全局样式文件
- **包含内容**：
  - 全局页面样式（page 选择器）
  - 全局组件样式
  - 全局公共样式

### 2. pages目录

pages目录用于存放所有页面，每个页面都是一个独立的文件夹，包含4个文件：

#### [页面名].js
- **作用**：页面的逻辑文件
- **包含内容**：
  - Page() 构造函数
  - 页面数据（data）
  - 生命周期函数（onLoad, onShow, onReady等）
  - 事件处理函数
  - 其他业务逻辑

#### [页面名].json
- **作用**：页面的配置文件
- **包含内容**：
  - 页面标题（navigationBarTitleText）
  - 是否启用下拉刷新（enablePullDownRefresh）
  - 页面窗口配置
  - 组件引用配置（usingComponents）

#### [页面名].wxml
- **作用**：页面的结构文件（类似HTML）
- **包含内容**：
  - 页面标签结构
  - 数据绑定
  - 事件绑定
  - 条件渲染
  - 列表渲染

#### [页面名].wxss
- **作用**：页面的样式文件（类似CSS）
- **包含内容**：
  - 页面样式定义
  - 组件样式
  - 选择器样式

## 三、最小项目需要哪些文件？

一个最基本的微信小程序至少需要以下文件：

1. **app.json** - 必须（定义页面路径）
2. **app.js** - 必须（使用App()函数）
3. 至少一个页面（包含 js, json, wxml, wxss 四个文件）

## 四、注意事项

1. **文件路径**：在配置文件中使用相对路径，以项目根目录为起点
2. **文件大小**：单个文件大小不能超过2MB
3. **编码格式**：建议使用UTF-8编码
4. **命名规范**：
   - 文件名使用小写字母
   - 组件名使用大驼峰命名（PascalCase）
   - CSS类名使用小写字母和中划线（kebab-case）
